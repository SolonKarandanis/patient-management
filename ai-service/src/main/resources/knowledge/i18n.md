# Resource Bundles (Translations / i18n)

## What are resource bundles?
Resource bundles are the translation labels used throughout the application — button text, form labels, error messages, and UI strings. They are stored in the database and can be managed directly from the application without redeploying. The feature is also called i18n management or translation management.

## How to access resource bundle management
Navigate to the Resource Bundles section from the left sidebar. If you cannot see this option, you either do not have the MANAGE_RESOURCE_BUNDLES permission or the feature has been disabled by the administrator. Ask your administrator to grant you the permission or enable the feature.

## Searching for resource bundle entries
On the Resource Bundles page you can search for translation entries using three filters: Language, Module, and a free-text Term. You do not need to fill in all filters. Select a language from the dropdown to narrow results to a specific language. Select a module to filter by application area. Enter a keyword in the Term field to search by translation key or text value. Click Search to execute the query. Results appear in the table below.

## Understanding the results table
Each row in the results table represents one translation key. Columns show the key name, the module it belongs to, and the translated text for each available language. If a translation is missing for a language the cell will be empty.

## How to edit a translation
Click the edit icon on the row you want to change. The row becomes editable inline. Modify the translation text for one or more languages. Click Save to persist the changes. The updated translation will take effect immediately for all users without requiring a page refresh or redeployment.

## Languages
Languages available in the system are managed by the administrator. Each language has an ISO code (for example: en for English, el for Greek) and a display label. You can filter resource bundle searches by language using the Language dropdown.

## Modules
Translations are grouped into modules that correspond to different areas of the application. Examples include UI labels, error messages, and validation messages. Use the Module filter on the search page to narrow results to a specific area.

## Enabling or disabling DB-backed translations
By default the application reads translations from static property files. When the administrator enables the database-backed translation mode (i18n.resources.DB.enabled=true) the application reads translations from the database instead. This allows translations to be updated live without redeployment. When this mode is disabled the Resource Bundles menu item does not appear even if you have the permission.
